# Changelog

All notable changes to the Folio Portfolio Template will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.3.0] - 2024-12-21

### Added
- **Dark/Light Mode Toggle** - Complete theme switching system with smooth transitions
- **CSS Custom Properties** - Comprehensive variable system for easy theme customization
- **Theme Persistence** - User preferences saved to localStorage
- **Open Graph Meta Tags** - Social media sharing optimization for Facebook and Twitter
- **Enhanced Theme Button** - Animated toggle button with icon transitions
- **Dark Mode Styles** - Complete dark theme implementation for all sections

### Changed
- Converted all hardcoded colors to CSS custom properties
- Enhanced navbar with theme toggle integration
- Updated all sections to support both light and dark themes
- Improved theme transition animations
- Updated documentation with dark mode customization guide

### Fixed
- Theme consistency across all components
- Improved accessibility with proper ARIA labels
- Enhanced mobile theme toggle experience
- Fixed theme persistence across page reloads

## [1.2.0] - 2024-12-20

### Changed
- Updated color scheme to modern blue/purple/teal palette
- Improved typography with Inter font family
- Enhanced button hover effects and animations
- Optimized CSS for better performance
- Updated portfolio section with glass-morphism effects
- Improved mobile layout and spacing

### Fixed
- Fixed mobile navigation positioning

- Fixed portfolio filter button styling
- Fixed responsive grid layouts
- Fixed form input styling on mobile devices

## Upgrade Instructions

### From v1.1.0 to v1.2.0
1. Backup your current files
2. Replace `style.css` with the new version
3. Replace `script.js` with the new version
4. Update any custom CSS to match new color scheme
5. Test responsive design on all devices

### From v1.0.0 to v1.1.0
1. Backup your current files
2. Replace all files with new versions
3. Update your content in `index.html`
4. Test all functionality
5. Verify responsive design

---

**Note**: Always backup your customized files before updating to a new version.
